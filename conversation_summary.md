# 对话总结

## 1. 训练阶段基于训练集确定测试阈值
- 当前做法是在训练阶段通过 `compute_epoch_centers(...)` 计算每个已知类的中心 `centers`。
- 讨论的目标是在此基础上，进一步统计训练样本到所属类中心的余弦距离分布，并据此为测试阶段提供拒识阈值。
- 最终收敛的方案是：
  - 使用余弦距离 `dist = 1 - cosine_similarity`
  - 阈值定义为训练集距离分布的 `95%` 分位点
  - 阈值粒度先采用“全局一个阈值”，而不是“每类一个阈值”
- 这意味着测试阶段不再使用固定常数阈值，而是使用训练集自适应得到的阈值。

## 2. centers 做两次归一化是否有影响
- 结论：基本没有实质影响。
- 如果 `centers` 已经是 L2 归一化后的单位向量，再次归一化数学上结果不变。
- 实际影响主要只有：
  - 极小的浮点误差
  - 从 `csv` 读回时可修正文本截断带来的范数偏差
- 更关键的问题不是中心是否重复归一化，而是：
  - 计算类中心时，是否应先对样本特征 `feats` 做 L2 归一化
  - 这会影响“类中心”的定义是否与测试时的余弦距离判别保持一致

## 3. Git 工作流中是否一直不需要 rebase
- 结论：`rebase` 是否需要，与使用 `mac` 还是 `win` 无关，只与 Git 分支历史是否分叉有关。
- 如果流程一直是：
  - 在 `mac` 上修改和提交代码
  - 推送到远端
  - `win` 上只拉代码运行，不产生新的提交
- 那么通常不需要频繁 `rebase`。
- 只有在以下情况才需要考虑 `rebase`：
  - `win` 上也产生了新提交
  - 其他人向同一分支推送了提交
  - 本地分支落后于远端且希望保持线性历史

## 4. 椭圆阈值是否适合当前余弦距离方案
- 结论：对当前“L2 归一化特征 + 余弦距离”的方案，不适合直接照搬椭圆阈值。
- 原因是：
  - 椭圆阈值本质是欧氏空间中的二次型边界
  - 当前方法关注的是角度关系而不是欧氏长度
  - 特征归一化后落在单位球面上，“等余弦距离”的边界更接近球冠或角度锥，而不是欧氏椭圆
- 更合理的方向是：
  - 全局一个余弦距离阈值
  - 每类一个余弦距离阈值
  - 更进一步可做球面分布建模（如 vMF），而不是椭圆建模

## 5. radar 论文的阅读结论
论文：
`Radar_HRRP_Open_Set_Recognition_Based_on_Extreme_Value_Distribution.pdf`

核心结论：
- 论文关注的是 open set recognition 中“已知类簇的闭集边界”。
- 它的核心思想不是分类分数阈值，而是：
  - 在训练后得到已知类的特征簇
  - 找到每个簇的中心
  - 研究训练样本到类中心的距离分布
  - 利用极值理论（EVT/GEV）描述类边界
- 论文强调的是：
  - open set recognition 的本质是寻找已知类簇的闭边界
  - 样本越靠近类中心，越可能属于该类
  - 边界点可以看作“极值”，适合用极值分布拟合

这和当前思路是同方向的：
- 当前也在做“样本到类中心的距离”
- 只是当前先采用经验分位点（95% 分位点）作为简化版本
- 论文则进一步使用 EVT/GEV 对尾部分布建模

## 6. 余弦距离是否适用于极值方法
- 结论：适用，而且比椭圆阈值更匹配当前方案。
- 极值方法真正需要建模的是一个标量随机变量的尾部。
- 在论文里，这个标量是 `dist(X, O)`，即样本到类中心的距离。
- 这个距离不要求必须是欧氏距离，只要能稳定刻画“离类中心有多远”即可。
- 因此当前定义：
  - `dist = 1 - cosine_similarity`
  是可以直接作为极值建模对象的。
- 更合理的做法是：
  - 对训练样本计算“到真实类别中心的余弦距离”
  - 再对这些距离的尾部进行建模
- 由于余弦距离在归一化后有上界，理论上更接近“有限上界”的极值情形，因此与 EVT 并不冲突。

## 7. OpenMax 源码与核心机制
找到的 OpenMax 官方源码仓库：
- `abhijitbendale/OSDN`

核心机制总结：
- OpenMax 先在训练集上统计每个类的 `MAV`（Mean Activation Vector）
- 再对训练样本到对应 `MAV` 的距离分布尾部做 Weibull 拟合
- 测试时，对 top-k 类的激活进行重校准：
  - 离类 `MAV` 越远，说明越不像该类
  - 该类激活会被削减
  - 被削减的部分分配给 `unknown`
- 最后输出带有 `unknown` 类的 OpenMax 概率

补充结论：
- OpenMax 官方实现中支持多种距离，包括 `cosine`
- 因此 OpenMax 不是只能用欧氏距离

## 8. MAV 与 embedding 的区别
- 结论：OpenMax 的 `MAV` 不是当前意义上的 embedding center。
- `MAV` 是分类层前后接近决策空间的激活均值，通常对应 `logit/activation` 空间。
- 它还不是 softmax 概率，但已经非常接近分类决策空间。
- 当前方法中的 `center` 则是 feature/embedding space 的类中心。
- 因而两条路线不同：
  - 当前方法：在 embedding 空间中做“类簇边界”
  - OpenMax：在 activation/logit 空间中做“分类证据重校准”

## 9. radar 论文是否是在 embedding 上做的
- 结论：是。
- radar 论文中的对象是训练后得到的 `features`
- 同类 `features` 在 feature space 形成簇
- 然后研究样本到类中心 `O` 的距离分布
- 因此它属于 embedding/feature space 上的边界建模方法
- 它不是 OpenMax 那种在 `MAV/logit` 空间做校准的方法

## 10. 整体结论
- 当前最合理的路线是继续沿着 radar 论文这条线走：
  - 在 embedding space 中构造类中心
  - 用余弦距离度量样本到类中心的远近
  - 用训练集距离分布确定测试拒识阈值
- 现阶段最简洁可行的实现是：
  - 使用训练集到类中心余弦距离的 `95%` 分位点作为全局阈值
- 如果后续要进一步贴近 radar 论文，可以升级为：
  - 每类独立阈值
  - 每类尾部分布的 EVT/GEV 拟合
- 不建议在当前余弦距离框架下直接改用椭圆阈值。
